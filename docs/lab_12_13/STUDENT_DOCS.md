# STUDENT_DOCS

## Тема

Розробка вебзастосунку інтернет-магазину товарів для домашніх тварин `Petizoo`

## Вступ

У сучасних умовах розвиток електронної комерції є одним із найважливіших напрямів цифровізації бізнесу. Все більше користувачів віддають перевагу онлайн-замовленням, оскільки це дозволяє економити час, швидко порівнювати товари та оформлювати покупки без відвідування фізичних магазинів. Особливо актуальним це є для сфери продажу товарів для домашніх тварин, де покупець часто потребує регулярного придбання корму, аксесуарів, засобів гігієни та інших супутніх товарів.

Метою даної роботи є аналіз та опис вебзастосунку `Petizoo`, який реалізує основний функціонал інтернет-магазину товарів для котів і собак.

Для досягнення поставленої мети необхідно виконати такі завдання:

- проаналізувати предметну область онлайн-продажу товарів для домашніх тварин;
- визначити основні функціональні можливості вебзастосунку;
- дослідити технологічний стек, який використовується в проєкті;
- описати структуру та принцип роботи системи;
- обґрунтувати практичну значущість реалізованого рішення.

Об’єктом дослідження є процес онлайн-продажу товарів для домашніх тварин.

Предметом дослідження є методи, засоби та програмні технології розробки вебзастосунку інтернет-магазину `Petizoo`.

Актуальність теми полягає в тому, що онлайн-торгівля стала звичним і зручним способом взаємодії між продавцем і покупцем. Для магазинів товарів для тварин наявність вебплатформи дозволяє розширити аудиторію, автоматизувати приймання замовлень, покращити представлення асортименту та підвищити якість обслуговування клієнтів. Крім того, створення подібного застосунку є актуальним з навчальної точки зору, оскільки дозволяє закріпити знання з вебпрограмування, роботи з базами даних, маршрутизації, обробки форм і організації архітектури вебсистеми.

Практичне значення роботи полягає в тому, що створений застосунок може бути використаний як основа для реального інтернет-магазину або як база для подальшого розвитку дипломного проєкту. Реалізована система вже підтримує перегляд каталогу, пошук і фільтрацію товарів, роботу з кошиком, оформлення замовлення, особистий кабінет користувача та адміністративні функції для керування товарами і замовленнями.

## Умовні позначення

- `MVC` - модель архітектури, у якій застосунок поділяється на Model, View, Controller.
- `PHP` - серверна мова програмування, яка використовується для реалізації логіки системи.
- `MySQL` - система керування базами даних, що використовується для зберігання даних.
- `HTTP` - протокол передачі даних між клієнтом і сервером.
- `AJAX` - підхід для асинхронного обміну даними без повного перезавантаження сторінки.
- `UI` - користувацький інтерфейс.
- `CRUD` - базові операції створення, читання, оновлення та видалення даних.

## Розділ 1. Опис технологічного стеку

Для розробки вебзастосунку `Petizoo` використано класичний стек вебтехнологій, який добре підходить для створення серверних інформаційних систем і навчальних проєктів.

Основною мовою програмування є `PHP`. Саме на PHP реалізовано маршрутизацію, контролери, обробку форм, логіку авторизації, оформлення замовлень, роботу з кошиком і взаємодію з базою даних. У проєкті використано власну легку `MVC`-структуру, де окремо виділено моделі, представлення та контролери. Такий підхід покращує читабельність коду, дозволяє логічно розділити відповідальність між компонентами й спрощує подальше розширення системи.

Для зберігання даних застосовується `MySQL`. У базі даних зберігаються користувачі, товари, категорії, підкатегорії, кошик авторизованих користувачів, обрані товари, замовлення, позиції замовлень і коментарі. Використання реляційної бази даних є доцільним, оскільки структура інтернет-магазину містить чітко пов’язані сутності, між якими існують зрозумілі логічні зв’язки.

Для запуску проєкту в локальному середовищі використовується `MAMP`. Це зручно для розробки та тестування, оскільки середовище дозволяє швидко розгорнути вебсервер, PHP та MySQL на локальному комп’ютері без складного ручного налаштування.

Клієнтська частина реалізована за допомогою `HTML`, `CSS` та стандартних можливостей браузера. `HTML` використовується для побудови структури сторінок, форм, таблиць і блоків контенту. `CSS` відповідає за оформлення сторінок, зовнішній вигляд каталогу товарів, кошика, сторінок профілю та адміністративних екранів. Такий підхід дозволяє створити зрозумілий інтерфейс і забезпечити базову зручність користування системою.

У деяких частинах проєкту використовується `AJAX`, зокрема для роботи з кошиком і списком обраного без повного оновлення сторінки. Це покращує взаємодію з інтерфейсом і робить застосунок більш зручним для кінцевого користувача.

Для роботи з користувачами в системі застосовується механізм сесій. Після успішної авторизації дані користувача зберігаються в сесії, що дозволяє підтримувати стан входу, працювати з персональним кабінетом, обраними товарами та кошиком. Для безпеки облікових даних паролі в системі зберігаються у хешованому вигляді.

Окремо варто відзначити файлову структуру проєкту. Папки `controllers`, `models`, `views`, `core`, `css`, `public` та `uploads` формують логічний поділ системи на бізнес-логіку, відображення, інфраструктурну частину та статичні ресурси. Така організація є зрозумілою, зручною в підтримці та відповідає навчальним цілям курсового вебпроєкту.

Отже, обраний технологічний стек є достатнім для реалізації інтернет-магазину середнього рівня складності, добре підходить для навчального проєкту та може бути розширений у майбутньому за рахунок підключення нових модулів, API або сучасніших фронтенд-рішень.

## Висновки

У межах даної роботи було розглянуто вебзастосунок `Petizoo` як приклад реалізації інтернет-магазину товарів для домашніх тварин. У процесі підготовки документа було визначено мету, завдання, об’єкт і предмет дослідження, а також обґрунтовано актуальність теми та її практичне значення.

Було встановлено, що реалізований застосунок охоплює ключові бізнес-процеси онлайн-магазину: перегляд каталогу, фільтрацію товарів, реєстрацію та авторизацію користувачів, керування кошиком, оформлення замовлень, збереження обраних товарів і адміністрування асортименту. Це свідчить про те, що проєкт має завершену базову структуру та може розглядатися як повноцінна основа для подальшого вдосконалення.

Проаналізований технологічний стек показав, що використання PHP, MySQL, HTML, CSS, AJAX і локального середовища MAMP є доцільним для реалізації подібної системи. Такий набір технологій забезпечує зрозумілу архітектуру, стабільну роботу й можливість подальшого масштабування.

Таким чином, розроблений проєкт має не лише навчальну, а й прикладну цінність, оскільки може бути використаний як основа для реального вебресурсу або як база для майбутньої дипломної роботи.
